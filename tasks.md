

## 🏗️ فاز ۱: معماری پایه، قالب‌بندی و تسلط بر TypeScript
**هدف:** تنظیم پروژه، ساختار پوشه‌بندی استاندارد و قوی کردن پایه‌های TypeScript در محیط React/Next.js.

- [ ] **تسک ۱.۱: راه‌اندازی پروژه و ساختار معماری (Folder Structure)**
  - [ ] ساب‌تاسک: ایجاد پروژه جدید Next.js با استفاده از App Router و TypeScript.
  - [ ] ساب‌تاسک: پیکربندی Tailwind CSS و نصب کتابخانه Shadcn UI.
  - [ ] ساب‌تاسک: ایجاد پوشه‌بندی استاندارد شرکتی در دایرکتوری `src` (شامل `components`, `hooks`, `types`, `services`, `context`, `store`).

- [ ] **تسک ۱.۲: طراحی صفحه محصولات و حل چالش TypeScript**
  - [ ] ساب‌تاسک: تعریف یک فایل اختصاصی برای تایپ‌ها (`src/types/product.ts`).
  - [ ] ساب‌تاسک: ساخت اینترفیس `Product` شامل فیلدهای: `id`, `title`, `price`, `description`, `category`, `image`, `rating`.
  - [ ] ساب‌تاسک: ساخت کامپوننت `ProductCard` با استفاده از Shadcn و Tailwind.
  - [ ] ساب‌تاسک: اتصال دقیق تایپ `Product` به عنوان Props ورودی به کامپوننت `ProductCard` و رفع تمام خطاهای کامپایلر لایه تایپ‌اسکریپت.

## 🛒 فاز ۲: سیستم سبد خرید و مدیریت وضعیت (State Management)
**هدف:** درک عمیق مدیریت وضعیت در سطوح مختلف؛ چه زمانی از Context استفاده کنیم و چه زمانی به سراغ Zustand برویم.

- [ ] **تسک ۲.۱: پیاده‌سازی ویژگی‌های ظاهری و وضعیت‌های ساده با Context API**
  - [ ] ساب‌تاسک: ساخت یک سیستم تغییر تم (Dark/Light Mode) یا مدیریت سایدبار با استفاده از Context API خود ری‌آکت.
  - [ ] ساب‌تاسک: درک تفاوت دیتای محلی کامپوننت با دیتای اشتراکی در Context و چگونگی جلوگیری از Prop Drilling.

- [ ] **تسک ۲.۲: پیاده‌سازی سبد خرید پیشرفته با Zustand**
  - [ ] ساب‌تاسک: نصب Zustand و ایجاد اولین استور جهانی به نام `useCartStore`.
  - [ ] ساب‌تاسک: تعریف اینترفیس‌های مربوط به استور سبد خرید (`CartItem` که ارث‌بری دارد از `Product` به همراه فیلد `quantity`).
  - [ ] ساب‌تاسک: پیاده‌سازی اکشن‌های `addToCart` (با منطق اضافه کردن تعداد در صورت وجود محصول).
  - [ ] ساب‌تاسک: پیاده‌سازی اکشن‌های `removeFromCart`، `clearCart` و آپدیت تعداد محصول.
  - [ ] ساب‌تاسک: اتصال تعداد کل محصولات سبد خرید به هدر سایت (Navbar) به صورت زنده.
  - [ ] ساب‌تاسک: ذخیره‌سازی وضعیت سبد خرید در LocalStorage با استفاده از Middleware پpersist در Zustand تا با رفرش صفحه پاک نشود.

## 🔐 فاز ۳: احراز هویت و مدیریت امن توکن‌ها (Authentication)
**هدف:** یادگیری سیستم‌های امنیتی فرانت‌اند، کار با JWT، کوکی‌ها و محافظت از صفحات (Route Protection).

- [ ] **تسک ۳.۱: ساخت فرم و درک مکانیزم JWT**
  - [ ] ساب‌تاسک: طراحی فرم لاگین با استفاده از ابزار `Form` و `Input` در Shadcn به همراه ولیدیشن‌های اولیه.
  - [ ] ساب‌تاسک: مطالعه و درک تفاوت Access Token و Refresh Token و نحوه کارکرد آن‌ها در لایه کلاینت و سرور.

- [ ] **تسک ۳.۲: پیاده‌سازی احراز هویت با NextAuth.js (Auth.js)**
  - [ ] ساب‌تاسک: نصب و کانفیگ NextAuth در Next.js App Router (پیکربندی Credentials Provider).
  - [ ] ساب‌تاسک: مدیریت ذخیره‌سازی توکن‌ها در Session و انتقال امن آن به کوکی‌های مرورگر به صورت HttpOnly.
  - [ ] ساب‌تاسک: پیاده‌سازی کامپوننت‌های شرطی (نمایش دکمه "پروفایل" در صورت لاگین بودن و دکمه "ورود" در صورت مهمان بودن).
  - [ ] ساب‌تاسک: ساخت یک `middleware.ts` در ریشه پروژه برای محافظت از صفحات پنل کاربری و تسویه حساب (Checkout)، به طوری که کاربر غیر لاگین خودکار به صفحه `/login` ریدایرکت شود.

## 🌐 فاز ۴: ارتباط با API، کشینگ و هوک‌های پرفورمنس
**هدف:** جایگزین کردن روش‌های سنتی فچ داتا با ابزارهای مدرن صنعت وب، مدیریت لودینگ/خطا و بهینه‌سازی رندرهای ری‌آکت.

- [ ] **تسک ۴.۱: مدیریت داتا با TanStack Query (React Query)**
  - [ ] ساب‌تاسک: نصب و راه‌اندازی `QueryClientProvider` در پروژه‌.
  - [ ] ساب‌تاسک: پیاده‌سازی متدهای فچ کردن محصولات از یک API واقعی (مانند FakeStoreAPI) با استفاده از `useQuery`.
  - [ ] ساب‌تاسک: مدیریت وضعیت‌های هندلینگ خودکار لودینگ (`isLoading`) و خطا (`isError`) با نمایش کامپوننت Skeleton یا تستر خطای Shadcn.
  - [ ] ساب‌تاسک: پیاده‌سازی سیستم سرچ و فیلتر محصولات بر اساس دسته‌بندی با استفاده از سینک کردن URL Query Params در کامپوننت کلینت.

- [ ] **تسک ۴.۲: بهینه‌سازی پرفورمنس اپلیکیشن با هوک‌های پیشرفته**
  - [ ] ساب‌تاسک: استفاده از هوک `useMemo` برای محاسبه قیمت کل سبد خرید و تخفیف‌ها تا محاسبات فقط با تغییر آیتم‌های سبد خرید تکرار شوند.
  - [ ] ساب‌تاسک: استفاده از هوک `useCallback` برای توابعی که به کامپوننت‌های فرزند پاس داده می‌شوند تا از رندرهای مجدد بی‌مورد (Re-rendering) جلوگیری شود.
  - [ ] ساب‌تاسک: تایپ‌ریزی دقیق برای تمام رویدادهای مرسوم ری‌آکت مثل `ChangeEvent<HTMLInputElement>` در فرم‌ها و سرچ‌بار.
